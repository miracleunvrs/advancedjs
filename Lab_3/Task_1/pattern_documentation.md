# Документация паттерна: deepClone Pattern
**Pattern Name:** deepClone Pattern
**Description:** Работа с иммутабельными данными в JavaScript, где объекты передаются по ссылке.

# Проблема:
- Поверхностное копирование не копирует вложенные объекты, что приводит к нежелательным изменениям оригинала

# Решение:
- Рекурсивный обход всех свойств объекта и создания новых экземпляров для каждого уровня вложенности

# Дизайн:
- **Абстракция дизайна:** Паттерн предоставляет одну простую функцию, скрывая внутри сложную логику рекурсии и проверок типов. Это обеспечивает «прозрачность паттерна» (Pattern Transparency).

- **Рекурсивная стратегия:** Дизайн предусматривает разделение задачи на подзадачи. Если текущий узел — примитив, он возвращается. Если узел — объект, создается новый контейнер, и процесс запускается заново для всех его свойств

- **Иммутабельность:** Основная цель дизайна — поддержка концепции неизменяемости данных. После клонирования любые изменения в новом объекте не должны влиять на состояние системы, связанное с исходным объектом.
