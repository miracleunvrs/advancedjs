# Lab 6.3 - Task 3
## Decision Journal

**Chosen scenarios:** SCENARIO_01 и SCENARIO_02 - наиболее очевидные по признакам.

**Key clues:**
- единственный экземпляр -> Creational
- уведомления -> Behavioral
- смена алгоритма -> Behavioral
- упрощенный интерфейс -> Structural
- пошаговая сборка -> Creational
- запрос как объект -> Behavioral

**Main takeaway:**
Сначала нужно определить категорию паттерна,
затем выбирать конкретный.
Это ускоряет архитектурные решения и снижает риск ошибок.
