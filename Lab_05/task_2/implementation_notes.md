# Особенности реализации (Implementation Notes)

## Статические импорты (Static Imports)
В файле `app.mjs` мы использовали статические импорты для:
- `constants.mjs` (Константы)
- `core.mjs` (Базовые вычисления)
- `utils.mjs` (Работа с DOM)

**Почему:** Эти модули критически важны для работы приложения при запуске. Без них калькулятор не сможет даже отобразить название или выполнить простое сложение. Они загружаются сразу (eagerly).

## Динамические импорты (Dynamic Imports)

### 1. Import on Interaction (advanced_feature.mjs)
**Сценарий:** Загрузка "Инженерного режима".
**Почему:** Функции вроде `sqrt` (квадратный корень) нужны не всем пользователям. Мы загружаем этот код только по клику на кнопку. Это уменьшает объем данных, которые браузер должен скачать при первой загрузке страницы.

### 2. Import on Visibility (lazy_component.mjs)
**Сценарий:** Загрузка истории операций.
**Почему:** Блок истории находится внизу страницы. Если пользователь не прокрутит страницу до него, код для отрисовки истории никогда не будет скачан. Это оптимизирует производительность.

## Обработка ошибок
Все динамические импорты обернуты в блоки `try...catch`. Если модуль не удастся загрузить (например, из-за проблем с сетью), приложение не "сломается", а выведет ошибку в консоль и предложит запасной вариант.
