# Документация по модернизации кода (Lab 5.1)

## 1. Процесс рефакторинга и модульность
Исходный монолитный код был разделен на 4 независимых модуля для обеспечения принципа **Decoupling** (разделение ответственности):

*   **`config.mjs`**: Содержит константы приложения (`taxRate`, `currency`). Это позволяет изменять глобальные настройки без вмешательства в логику.
*   **`utils.mjs`**: Модуль вспомогательных функций. Реализован паттерн **Module Object** (`import * as config`), что группирует все настройки в один объект.
*   **`core.mjs`**: Содержит основные классы. Это "сердце" системы, полностью независимое от способов отображения данных.
*   **`main.mjs`**: Точка входа, которая оркестрирует работу остальных модулей.

## 2. Примененные современные стандарты (ES6+)

### Замена `var` на `const/let`
Все объявления `var` были заменены на `const` (для объектов и констант) и `let` (для счетчиков). 
*   **Результат:** Устранена проблема "всплытия" (hoisting) и обеспечена блочная область видимости, что делает код безопаснее.

### Переход к классам (ES6 Classes)
Функции-конструкторы `Product` и `Cart` были преобразованы в полноценные классы.
*   **Результат:** Код стал более структурированным. Методы теперь находятся внутри блока класса, а не разбросаны по прототипам.

### Инкапсуляция и приватные поля
Использован современный синтаксис приватных полей с префиксом `#` (например, `#items` в классе `Cart`).
*   **Результат:** Реализована истинная инкапсуляция. Внешний код больше не может напрямую изменять массив товаров или цену продукта, что гарантирует целостность данных. Доступ к данным реализован через **геттеры**.

### Шаблонные строки (Template Literals)
Вместо конкатенации строк через оператор `+` использованы обратные кавычки и синтаксис `${}`.
*   **Результат:** Улучшена читаемость кода, особенно в функциях форматирования вывода.

### Стрелочные функции (Arrow Functions)
Функции утилит преобразованы в стрелочные.
*   **Результат:** Более лаконичный синтаксис.

## 3. Выгоды от модернизации
1.  **Масштабируемость:** Благодаря модульной структуре, в проект легко добавить новые типы товаров или новые налоги, не затрагивая старый код.
2.  **Читаемость:** Современный синтаксис (классы, шаблоны) делает код понятным для других разработчиков "с первого взгляда".
3.  **Безопасность:** Приватные поля защищают логику приложения от случайных или намеренных ошибок при манипуляции данными извне.
4.  **Совместимость:** Использование расширения `.mjs` и `"type": "module"` в `package.json` обеспечивает нативную поддержку современной модульной системы Node.js.
